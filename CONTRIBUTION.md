## 🔄 协作流程

### 版本控制

```
main 分支
  ├── feature/data-analysis (成员2)
  ├── feature/ai-integration (成员3)
  ├── feature/frontend (成员4)
  └── feature/testing (成员5)
```

### 代码审查

- 所有功能分支合并前需要至少 1 人审查
- 重要功能由组长进行最终审核
- 使用 Pull Request 进行代码合并

### 沟通方式

- 每周 1-2 次线上/线下会议
- 使用微信群进行日常沟通
- 使用 GitHub Issues 跟踪任务
