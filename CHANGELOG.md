## [1.1.0] - 2025-11-27

### 🎉 目录结构优化

#### ✨ 新增
- 创建 `data/` 目录，分离原始数据和处理后数据
  - `data/raw/` - 存放原始 Excel 文件
  - `data/processed/` - 存放处理后的 CSV 文件
- 创建 `docs/` 目录，集中管理所有文档
  - 快速启动.md
  - 使用指南.md
  - 项目说明.md
  - 目录结构说明.md
- 新增 `.gitignore` 文件
- 新增 `CHANGELOG.md` 更新日志

#### 🔄 修改
- 更新 `config.json` 中的 `data_directory` 为 `data/processed`
- 更新 `process_data.py` 中的文件路径
- 更新 `start.sh` 中的数据文件检查路径
- 重写根目录 `README.md`，使其更简洁

#### 📁 目录结构对比

**优化前：**
```
python_house1/
├── app.py
├── config.json
├── process_data.py
├── *.xlsx (原始数据，3个文件)
├── data_*.csv (处理后数据，3个文件)
├── README.md
├── 使用指南.md
├── 项目说明.md
├── 快速启动.md
├── templates/
├── static/
└── venv/
```

**优化后：**
```
python_house1/
├── app.py
├── config.json
├── process_data.py
├── requirements.txt
├── start.sh
├── .gitignore
├── README.md
├── CHANGELOG.md
├── data/
│   ├── raw/          (原始 Excel)
│   └── processed/    (处理后 CSV)
├── templates/
├── static/
├── docs/             (所有文档)
└── venv/
```

#### ✅ 优势
1. **数据分离**：原始数据和处理后数据清晰分开
2. **文档集中**：所有文档在 `docs/` 目录下，易于查找
3. **结构清晰**：根目录只保留核心文件，更加整洁
4. **便于管理**：.gitignore 规范了版本控制
5. **易于扩展**：新增城市数据只需放入对应目录
